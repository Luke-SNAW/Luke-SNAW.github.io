What is Functional Composition

Composition is the process of composing small units into bigger units that solve bigger problems. Where Input of one function comes from the output of previous one.

For example, if we want to build a castle üè∞ we‚Äôll have to compose bricks üß±

Examples:

Note: as a personal preference I‚Äôll be using pipe in the examples.

Let‚Äôs say we need to build a simple price calculator, where we can apply:

  1. Discount
  2. Coupon
  3. Tax (default = 30%)
  4. Service fees (default = 10)
  5. Weight-based shipping cost

Let‚Äôs first design the API of the price calculator based on the given requirements

  price,
  discount,
  percentCoupon,
  valueCoupon,
  weight,
  $PerKg

We defaulted tax to 30% and serviceFees to 10$. Waiting for the rest of the params

The bad way (non compositional)

Let‚Äôs build it as an atomic mathematical equation:

const priceCalculator = (
  taxPercentage = 0.3,
  serviceFees = 10,
  price,
  discount,
  percentCoupon,
  valueCoupon,
  weight,
  $PerKg
) => {
  return (
    (price -
      price * percentCoupon -
      discount -
      couponValue +
      weight * $PerKg +
      serviceFees) *
    (1 + taxPercentage)
  );
};

It does the job. But it has very poor reading, testing, debugging and maintenance experiences.

Let‚Äôs do it the good way (composition approach)

Let‚Äôs compose‚Ä¶

const priceCalculator = (
  taxPercentage = 0.3,
  serviceFees = 10,
  price,
  discount,
  percentCoupon,
  valueCoupon,
  weight,
  $PerKg
) => {
  const applyTax = (val) => val * (1 + taxPercentage);
  const applyServiceFees = (val) => val + serviceFees;
  const applyPercentCoupon = (val) => val - val * percentCoupon;
  const applyValueCoupon = (val) => val - valueCoupon;
  const applyDiscount = (val) => val - discount;
  const applyShippingCost = (val) => val + weight * $PerKg;

  return pipe(
    applyPercentCoupon,
    applyDiscount,
    applyValueCoupon,
    applyShippingCost,
    applyServiceFees,
    applyTax
  )(price);
};

This looks so much cleaner, more testable, debuggable and maintainable. All due to the modular mindset we‚Äôre adapting to.

We split each operation to be living on its own, then we pipe them and apply price to the piped functions.

Give it a try

priceCalculator(10); // NaN

We got a NaN and that‚Äôs unexpected, right?! How can we fix this?

Let‚Äôs debug first

Let me introduce a very useful utility to debug chains (pipe and compose); inspect function. It‚Äôs very simple. It only logs what it gets, and returns it.

const inspect = (tag) => (x) => {
  console.log(`${tag}: ${x}`);
  return x;
};

Now let‚Äôs add that to our chain of functions

const priceCalculator = (
  taxPercentage = 0.3,
  serviceFees = 10,
  price,
  discount,
  percentCoupon,
  valueCoupon,
  weight,
  $PerKg
) => {
  const applyTax = (val) => val * (1 + taxPercentage);
  const applyServiceFees = (val) => val + serviceFees;
  const applyPercentCoupon = (val) => val - val * percentCoupon;
  const applyValueCoupon = (val) => val - valueCoupon;
  const applyDiscount = (val) => val - discount;
  const applyShippingCost = (val) => val + weight * $PerKg;

  return pipe(
    inspect("price"),
    applyPercentCoupon,
    inspect("after applyPercentCoupon"),
    applyDiscount,
    inspect("after applyDiscount"),
    applyValueCoupon,
    inspect("after applyValueCoupon"),
    applyShippingCost,
    inspect("after applyShippingCost"),
    applyServiceFees,
    inspect("after applyServiceFees"),
    applyTax
  )(price);
};
